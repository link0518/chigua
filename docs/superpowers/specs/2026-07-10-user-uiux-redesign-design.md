# 用户端 UI/UX 重构设计规格

日期：2026-07-10

状态：方案已确认，等待书面规格复核

关联任务：`.trellis/tasks/07-10-user-uiux-redesign/`

## 目标

在不改变现有业务能力、后端接口和匿名身份语义的前提下，重构 JX3 瓜田用户端核心体验，使其具备成熟内容社区应有的清晰信息架构、稳定响应式布局、完整交互状态和一致品牌语言。

本次重构不是单纯换配色或统一圆角，而是解决以下结构性问题：

* 首页“单帖 / 列表”两种浏览模式职责重叠，发现内容与沉浸阅读的边界不清。
* 手绘边框、叠影、旋转、胶带等装饰在大量组件上同时出现，品牌元素变成视觉噪音。
* 手机端存在横向溢出、操作区被截断、顶部入口不利于单手操作等问题。
* 评论、回复输入和其它弹层存在多层叠加，软键盘和系统返回行为复杂。
* 发帖能力完整，但编辑、标签、媒体、预览和提交缺少成熟的信息层级。
* 加载、空、错误、禁用、重复提交等状态没有形成统一规范。

## 范围

本规格覆盖用户端：

* 全局导航与响应式用户端外壳。
* 首页最新信息流、热门信息流和沉浸浏览上下文。
* 帖子卡片、点赞、点踩、收藏、分享、举报和删除申请入口。
* 桌面评论右侧抽屉、手机评论底部抽屉、评论回复与评论输入。
* 桌面与手机发帖体验。
* 搜索、收藏、通知、“我的”、商城入口、标签屏蔽和设置入口。
* 骨架、空状态、错误状态、重试、禁用和成功反馈。
* 手机安全区、Android 软键盘、系统返回键和触控尺寸。

不纳入本次重构：

* Wiki；继续维持独立 Nordic 视觉。
* 管理后台；继续维持独立任务型界面。
* 新增关注、私信、草稿箱、推荐算法或账号体系。
* 修改数据库结构、后端权限或 API 语义。
* 引入新的 UI 框架、动画库或状态管理依赖。

## 核心决策

### 视觉方向：手帐进化

保留暖色纸张、方格背景、贴纸、少量手绘线条和“瓜”品牌符号，但将其从“每个组件的默认样式”收敛为“品牌强调层”。

正文、表单、导航和状态反馈使用稳定的产品化视觉；粗黑框、硬叠影、随机旋转和胶带只在少数品牌节点出现。

### 浏览结构：信息流负责发现，沉浸上下文负责阅读

```text
首页均衡信息流
├─ 点击帖子正文 → 沉浸浏览 → 上一个 / 下一个瓜
├─ 点击评论     → 桌面右侧评论抽屉
│                 手机底部评论抽屉
└─ 点击投稿     → 桌面/手机独立发帖界面
```

首页取消“单帖 / 列表”切换。旧单帖模式的价值保留在沉浸浏览上下文中，不再与首页发现模式竞争。

### 交付方式：两期交付

第一期完成设计基础和“发现 → 互动/评论 → 发布”核心闭环；第二期统一搜索、收藏、通知、“我的”及相关设置。每一期均可独立构建、验收和回退，不长期维护同一路径的新旧两套交互。

两期分别编写和审查实现计划。书面规格通过后只制定第一期实现计划；第二期在第一期验收完成后单独规划，避免一个计划同时覆盖全部用户端页面。

## 信息架构

### 手机端

```text
┌────────────────────────┐
│ 瓜田 Logo          通知 │
│ 页面标题 / 内容分段     │
├────────────────────────┤
│                        │
│ 当前页面内容            │
│                        │
│                   [投稿]│
├────────────────────────┤
│ 首页  热门  搜索  我的   │
└────────────────────────┘
```

规则：

* 底部导航固定为：首页、热门、搜索、我的。
* 投稿使用右下拇指热区的浮动按钮，不占用第五个导航位。
* 通知位于顶部右侧并显示未读状态。
* 收藏、商城、标签屏蔽和设置等低频入口进入“我的”。
* 发帖和其它需要专注输入的页面隐藏全局底部导航，避免与键盘和提交操作冲突。
* 所有底部区域使用 `env(safe-area-inset-bottom)`，不以固定像素假设系统手势区。

### 桌面端

```text
┌──────────────────────────────────────────┐
│ Logo  最新 热门 搜索 收藏        通知 投稿 │
├──────────────────────────┬───────────────┤
│ 主信息流                  │ 正在热议       │
│ 约 720–760px 阅读宽度     │ 热门标签       │
│ 均衡密度帖子卡片          │ 瓜田公告       │
└──────────────────────────┴───────────────┘
```

规则：

* 页面内容最大宽度建议为 `1200–1280px`。
* 主信息流保持舒适阅读宽度，侧栏约 `280–320px`，间距约 `28–32px`。
* 侧栏只承载热议、热门标签和公告，不堆叠无业务价值的数据卡片。
* 当可用宽度不足时，侧栏自动下沉或隐藏；不得通过压缩主内容制造窄列。

## 视觉系统

### 颜色角色

第一版令牌采用以下语义角色：

| 角色 | 建议值 | 用途 |
|---|---:|---|
| `paper` | `#F7F3EA` | 页面底色 |
| `surface` | `#FFFEFB` | 主要内容表面 |
| `surface-muted` | `#F2EFE7` | 次级区域、引用与上下文 |
| `ink` | `#272721` | 主文字与强边界 |
| `text-muted` | `#706D67` | 时间、说明和弱操作 |
| `line` | `#D4D0C7` | 常规分割线与输入边界 |
| `coral` | `#E98E86` | 品牌主强调、选中状态 |
| `marker-green` | `#B8D5A7` | 当前导航、成功和轻强调 |
| `marker-blue` | `#9ED8ED` | 标签与信息提示 |
| `danger` | `#C76059` | 错误、危险操作 |
| `warning` | `#D38B45` | 警告与临界状态 |

春节主题只能覆盖这些语义令牌，不改变组件结构、层级或触控逻辑。

### 字体与排版

* 正文、按钮、表单和数字统一使用项目现有中文无衬线字体栈，不新增在线字体依赖。
* 品牌 Logo、空状态插图文字和少量栏目强调可以保留手绘感。
* 帖子没有新增标题字段；信息流摘要直接来自 Markdown 正文的首段或文本化摘要。
* 正文默认行高约 `1.65–1.75`，阅读列不超过约 65 个中文字符的视觉宽度。
* 数字计数使用等宽数字特性，减少点赞和评论数变化导致的跳动。

### 间距、圆角和边界

* 基础间距采用 `4 / 8 / 12 / 16 / 24 / 32 / 48` 级别。
* 常规控件圆角以 `8–12px` 为主，大抽屉和大容器使用 `18–24px`。
* 常规卡片使用浅色边界或背景层级，不默认使用粗黑框。
* 黑色手绘边界和 `2px` 硬叠影只用于 Logo、浮动投稿、关键选中态和少量品牌卡片。
* 禁止为每张帖子卡随机旋转，避免滚动列表产生视觉晃动。

### 图标与动效

* 新重构组件统一优先使用已安装的 Phosphor 图标，控制统一描边粗细；旧组件可在迁移期间保留 Lucide。
* 普通状态变化使用 `120–180ms`，抽屉和页面级过渡使用 `200–240ms`。
* 动画只使用 `transform` 和 `opacity`，不使用 `top/left/width/height` 做逐帧动画。
* 支持 `prefers-reduced-motion`，关闭非必要位移和弹性效果。
* 按下反馈以轻微缩放或位移为主，不使用大幅摇晃、旋转或礼花效果干扰高频操作。

## 首页与信息流

### 均衡信息密度

手机首屏目标约显示两条普通文本帖子。该目标是排版密度基准，不通过固定卡片高度强制实现；长正文、图片和大字号仍可自然增高。

帖子卡片结构：

```text
┌────────────────────────────┐
│ 作者身份     时间       标签 │
│                            │
│ Markdown 正文摘要           │
│ 可选媒体摘要                │
│                            │
│ 赞/踩  评论  收藏  分享/更多 │
└────────────────────────────┘
```

规则：

* 卡片正文摘要由现有 `content` 生成，不新增标题字段。
* 手机文本摘要默认限制为约 4–6 行；图片只展示首张或紧凑媒体摘要。
* 点击正文或媒体进入沉浸浏览；点击互动按钮只执行对应动作，不触发卡片跳转。
* 标签继续限制为每帖最多两个，并可进入现有标签搜索。
* 列表卡片不承载完整评论树和复杂编辑器。

### 沉浸浏览

* 保留现有 `/post/:id` 分享能力和评论定位能力。
* 展示完整 Markdown、图片、帖子动作和“上一个 / 下一个瓜”。
* 从列表进入时记录列表来源和位置；退出或返回后恢复原滚动位置。
* 沉浸浏览不重新引入首页“单帖 / 列表”切换开关。

## 帖子互动

### 点赞与点踩

* 点赞和点踩互斥；点击当前已选操作会取消。
* 前端乐观更新图标、颜色和计数。
* 请求失败时回滚到请求前状态，并在动作附近或 Toast 中提示失败。
* 同一帖子请求未完成时合并或暂时禁用重复动作，避免计数竞态。
* 选中态同时使用形状/填充和颜色，不只依赖颜色传达。

### 收藏、分享与更多

* 收藏即时切换，失败时回滚。
* 手机优先调用可用的系统分享；不支持时复制现有分享链接。
* 举报、删除申请和其它低频操作进入“更多”菜单。
* “更多”在手机使用底部动作面板，在桌面使用轻量菜单；危险操作与普通操作分组。
* 删除申请继续遵循现有作者身份和审核状态，不改变权限语义。

## 评论

评论内容和业务行为由一套共享的评论视图承担，桌面与手机只更换外层容器，避免维护两套评论逻辑。

### 桌面右侧抽屉

* 宽度建议为 `420–480px`，从信息流右侧进入。
* 背景信息流保留可见但弱化，关闭后恢复原滚动位置。
* 抽屉顶部展示评论数、关闭操作和当前帖子上下文摘要。
* 抽屉底部固定评论输入区，评论列表在中间独立滚动。
* `Esc`、浏览器返回和关闭按钮均可关闭抽屉。

### 手机底部抽屉

用户已确认继续保留当前“下方抽屉”基本方式，不改成依赖左上角返回的完整页面。

抽屉状态：

* 默认停靠点约为视口高度的 `55%`，保留原帖上下文。
* 上拉或点击展开进入约 `92–94%` 的近全屏状态。
* 从展开态下滑先回到 `55%`；从默认态继续下滑关闭。
* 系统返回键关闭抽屉；若软键盘正在显示，先由系统关闭键盘，再由下一次返回关闭抽屉。
* 抽屉内提供关闭按钮，但不要求用户触达页面左上角。
* 点击遮罩或下滑关闭时，若评论输入不为空，需要确认是否放弃内容。

软键盘规则：

* 输入框聚焦后，抽屉自动扩展到近全屏。
* 输入区根据 `visualViewport` 或等价能力跟随可见区域，不被软键盘遮挡。
* 键盘收起后恢复用户进入输入前的抽屉停靠点。
* 底部内边距包含系统安全区。

评论结构：

* 保留热门评论、普通评论、楼层、评论点赞、回复和评论定位。
* 回复目标持续显示在输入区附近，并提供就近取消。
* 只保留一层输入器；不再出现“评论抽屉 → 回复输入弹窗 → 其它弹窗”的多层嵌套。
* 空评论显示首评引导；加载失败在评论区内提供重试，不关闭整个抽屉。

### 路由与返回

评论抽屉状态应与现有帖子/评论 URL 能力兼容：

* 从通知或分享链接进入指定评论时自动打开对应评论容器并定位。
* 浏览器或 Android 系统返回先关闭评论容器，再返回上一页面。
* 关闭评论容器不刷新信息流，不丢失帖子位置和已加载数据。

## 发帖

### 手机单页沉浸编辑

手机不使用三步向导，也不在信息流内使用长文本发帖抽屉。

```text
┌────────────────────────┐
│ 关闭      匿名投稿      │
├────────────────────────┤
│ 完全匿名        0 / 2000│
│ Markdown 紧凑工具栏     │
│                        │
│ 正文编辑区              │
│                        │
│ 标签（最多 2 个）       │
│ 图片 / 表情             │
├────────────────────────┤
│       预览   匿名发布    │
└────────────────────────┘
```

规则：

* 正文编辑区占据主要空间，保留 CodeMirror 和 Markdown 快捷能力。
* 工具栏提供常用格式、图片、表情和“更多”；空间不足时横向滚动，不压缩为难点击的小图标。
* 标签和媒体区域按需展开，继续沿用最多两个标签及自定义标签校验。
* 底部固定“预览 / 匿名发布”，适配软键盘和安全区。
* 全局底部导航在编辑期间隐藏。
* 用户已有输入时关闭或系统返回，需要确认是否放弃。

### 桌面发帖

* 在宽屏采用编辑与预览并排布局；较窄桌面自动回退为编辑/预览分段切换。
* 标签、匿名说明和安全验证位于编辑区下方或右侧辅助区域。
* 发布按钮保持稳定位置，不随正文长度滚出可视范围。

### 上传、验证与提交

* 保留现有图片类型校验、Markdown 链接插入和粘贴图片能力。
* 每次上传显示“上传中 / 成功 / 失败”状态；若后端不能提供真实字节进度，不展示虚假百分比。
* Turnstile 继续在发布动作中执行，验证加载或失败使用就地说明。
* 内容为空、超出 2000 字、未选择标签等错误显示在对应区域附近。
* 发布期间禁用重复提交；失败后保留正文、标签和媒体引用。
* 发布成功进入新帖沉浸浏览上下文，并给出简短成功反馈。

## 第二期页面

### 搜索

* 顶部搜索输入与标签筛选形成一个明确查询区。
* 搜索结果复用统一帖子卡片，不维护另一套视觉结构。
* 长关键字、长标签和长链接不得制造横向滚动。
* 无结果、请求失败和继续加载分别显示独立状态。

### 收藏

* 复用统一帖子卡片和互动状态。
* 取消收藏后保留当前列表位置并给出明确反馈，避免列表无解释地跳动。
* 空收藏状态提供返回首页的明确入口。

### 通知

* 桌面使用与全局头部对齐的通知面板，手机使用可展开至近全屏的底部抽屉。
* 未读、已读、可跳转和仅展示结果的通知使用统一层级。
* 跳转到帖子或评论后，返回时恢复通知阅读上下文。

### 我的

* 作为手机端低频功能入口，收纳收藏、商城、标签屏蔽、设置和当前匿名身份相关状态。
* 商城能力和装备状态保持现有业务语义，只统一视觉与操作反馈。
* 避免在一个超长弹窗内继续堆叠多层子弹窗；复杂区域优先使用分段视图或抽屉内页面。

## 状态与错误处理

### 加载

* 首页使用与帖子卡片相同形状的骨架，不使用覆盖整屏的通用旋转图标。
* 评论使用评论行骨架；发帖标签单独使用标签骨架。
* 首屏加载不得造成页头、底部导航或内容宽度跳动。

### 空状态

* 首页无帖子：说明当前没有内容，并提供投稿入口。
* 评论为空：邀请发表第一条评论。
* 搜索无结果：保留查询条件并提供调整建议。
* 收藏为空：提供返回首页入口。
* 通知为空：显示安静状态，不使用错误语气。

### 错误

* 可恢复的数据加载错误在原区域就地显示并允许重试。
* 表单校验错误靠近对应字段展示，不只依赖 Toast。
* 不可恢复错误继续向上抛出或进入页面级错误状态，禁止静默吞没。
* 日志只记录关键请求、分支和异常，不在动画、滚动或高频渲染中记录。

## 组件与状态边界

本次重构需要做有针对性的组件拆分，但不进行无关目录迁移或全项目重写。

建议边界：

* 用户端外壳：桌面头部、手机底部导航、浮动投稿、安全区。
* 帖子展示：帖子卡片、完整帖子正文、媒体摘要、帖子动作。
* 评论内容：评论列表、评论项、回复、热门评论、评论输入。
* 评论容器：桌面右侧抽屉、手机底部抽屉；只负责布局、手势和返回行为。
* 发帖内容：Markdown 编辑、工具栏、标签选择、媒体状态、预览和提交状态。
* 状态组件：骨架、空状态、错误状态、内联错误和 Toast。

当前 `App.tsx`、`HomeView.tsx`、`CommentModal.tsx` 和 `UserMeModal.tsx` 体积较大。实施时只提取服务于本次重构的展示与交互边界，保留现有 Context、API 封装和业务数据流，禁止顺手迁移无关功能。

不新增状态管理库。全局业务数据继续由现有 `AppContext` 管理；抽屉停靠点、编辑/预览、菜单开关等纯 UI 状态保留在最接近使用处。

## 数据流与异常路径

### 首页

```text
进入首页
→ 加载帖子
→ 应用现有标签/关键词屏蔽
→ 渲染统一帖子卡片
→ 触底或用户操作继续加载
→ 成功追加 / 失败就地重试
```

### 帖子互动

```text
点击动作
→ 保存请求前状态
→ 乐观更新 UI
→ 调用现有 API
├─ 成功：确认服务端结果
└─ 失败：回滚状态 + 提示
```

### 评论

```text
点击评论
→ 记录信息流位置
→ 打开桌面抽屉或手机底部抽屉
→ 加载评论 / 定位目标评论
→ 输入、回复或点赞
→ 关闭容器
→ 恢复原信息流位置
```

### 发帖

```text
进入发帖
→ 编辑正文 / 标签 / 媒体
→ 本地校验
→ Turnstile 验证
→ 提交现有发布 API
├─ 成功：清空编辑状态 → 新帖沉浸浏览
└─ 失败：保留输入 → 就地提示 → 允许重试
```

## 响应式与无障碍

* 最低验收宽度为 `390px`，不得出现页面级横向滚动。
* 触控目标不小于 `44 × 44px`，相邻危险操作保持足够间距。
* 所有图标按钮提供可访问名称，不能仅靠图形表达含义。
* 键盘用户可以访问导航、帖子操作、菜单、抽屉和编辑器工具栏。
* 焦点进入抽屉后限制在抽屉内，关闭后返回触发按钮。
* 选中、错误和成功状态不能只依赖颜色。
* 图片保留有意义的替代文本；装饰图片不进入阅读顺序。
* Android 系统返回优先级：关闭键盘 → 关闭当前菜单/抽屉 → 返回上一浏览上下文。

## 性能约束

* 不新增大体积视觉依赖或运行时动画库。
* 装饰背景不得阻塞内容首屏；手机端降低或关闭非必要背景动画。
* 列表动画只作用于新增内容或状态变化，不为每次滚动重复播放。
* 保持图片懒加载和现有 Markdown 图片查看器能力。
* 避免因字体、吉祥物、骨架尺寸或抽屉挂载造成新的 CLS。

## 分期与实施顺序

### 第一期：核心闭环与设计基础

1. 建立用户端语义令牌、通用状态与基础交互样式。
2. 重构用户端外壳、桌面头部、手机底部导航和浮动投稿。
3. 重构首页均衡信息流、帖子卡片和沉浸浏览。
4. 重构点赞/点踩、收藏、分享和“更多”操作。
5. 提取共享评论内容，接入桌面右侧抽屉和手机底部抽屉。
6. 重构桌面双栏发帖与手机单页沉浸发帖。

### 第二期：次级路径统一

1. 搜索与标签筛选。
2. 收藏和通知。
3. “我的”、商城入口、标签屏蔽和设置。
4. 全量加载、空、错误和重试状态统一。
5. 删除不再使用的旧样式和旧首页模式分支。

每一步保持类型检查和构建可通过。回退以小批次提交为边界，不设置长期运行时双主题或双交互开关。

## 测试与验证

### 行为测试

需要覆盖：

* 点赞与点踩互斥、取消、请求失败回滚和重复点击。
* 收藏切换与失败回滚。
* 评论抽屉打开、两个停靠点、关闭、返回键、评论定位和滚动位置恢复。
* 软键盘显示时评论输入、回复取消和发送。
* 发帖正文、标签、图片失败、Turnstile 失败、重复提交和成功跳转。
* 标签/关键词屏蔽继续作用于信息流。

不需要为颜色值、类名、像素阴影或纯装饰动画编写琐碎测试。

### 响应式手动验收

至少检查：

* `390 × 844`：手机核心路径、底部安全区、评论抽屉和软键盘。
* `768 × 1024`：窄平板/横向内容布局。
* `1024 × 768`：小型笔记本，首屏导航和核心内容必须可见。
* `1440 × 1000`：桌面主信息流与侧栏比例。

## 验收标准

* `390px` 宽移动视口无页面级横向滚动，主要操作不被裁切。
* 首页默认使用均衡信息流，点击帖子可进入沉浸浏览并前后切换。
* 手机四栏导航和浮动投稿适配底部安全区与单手操作。
* 桌面侧栏不会挤压正文阅读宽度。
* 点赞/点踩、收藏、分享、举报、删除申请、评论和发帖现有行为不回归。
* 桌面评论右侧抽屉关闭后恢复信息流位置。
* 手机评论底部抽屉默认约 `55%`，可展开、下滑关闭，并适配软键盘。
* 手机发帖可在一个页面完成正文、标签、媒体、预览和发布。
* 加载、空、错误、禁用和重复提交状态均有明确 UI。
* 第一期完成“发现 → 互动/评论 → 发布”闭环；第二期完成全部用户端次级路径统一。
