# 09-帖子标签功能

## 目标

为帖子系统增加完整的标签能力：

- 发帖时可选择已有标签
- 发帖时可创建新标签
- 在帖子列表点击标签后，跳转到 `search` 页面展示该标签下全部帖子
- 后台系统设置可配置“默认帖子标签”

## 前端行为

### 投稿页（`SubmissionView`）

- 页面初始化时调用 `GET /api/posts/tags` 拉取后台配置的默认标签（含使用次数）
- 用户可在“已有标签”中点选/取消
- 用户可输入新标签并添加（仅当前帖子生效）
- 选择上限：最多 `2` 个标签
- 标签长度：单个标签最多 `6` 个字符
- 投稿页正文输入区使用专业 Markdown 编辑器内核，但保留现有手绘风格 UI 与“预览 / 上传图片 / 表情 / 标签 / 提交”布局
- 编辑器支持 Markdown 工具栏、列表/引用续写、`Tab / Shift+Tab` 缩进与常用快捷键；图片与表情会插入到当前光标位置
- 预览区继续沿用现有 Markdown 渲染逻辑，单次回车即可在预览中正确换行
- 发帖提交时把 `selectedTags` 透传到创建帖子接口

### 列表页标签跳转

- 首页（`HomeView`）点击标签，跳转到：`/search?tag=<标签>`
- 热榜页（`FeedView`）点击标签，跳转到：`/search?tag=<标签>`
- 搜索页结果中的标签可继续点击，按标签重新搜索

### 搜索页（`SearchView`）

- 支持 URL 参数：
  - `q`：关键字
  - `tag`：标签
  - `page`：页码（从 1 开始）
- 支持三种搜索模式：
  - 仅关键字
  - 仅标签
  - 关键字 + 标签（AND 条件）
- 分页切换会同步更新 URL
- 支持清除标签筛选
- 监听 `popstate`，确保浏览器前进/后退时搜索状态同步

## 后端接口变更

### `GET /api/posts/tags`

返回可选标签列表（仅后台默认标签，按后台设置顺序返回）：

```json
{
  "items": [
    { "name": "职场", "count": 12 },
    { "name": "校园", "count": 8 }
  ]
}
```

约束：

- 仅返回后台系统设置中的默认标签
- `count` 统计仅针对默认标签在已发布帖子中的出现次数
- 返回数量由 `limit` 控制，默认 `50`，最大 `200`

### `GET /api/posts/search`

新增 `tag` 查询参数：

- `q` 为空且 `tag` 存在：按标签搜索
- `q` 存在且 `tag` 为空：按关键字搜索
- `q` 和 `tag` 都存在：组合搜索

### `POST /api/posts`

新增标签清洗流程（`sanitizeTags`）：

- 非数组输入按空数组处理
- 去首尾空白
- 去掉前缀 `#`
- 多空白折叠成单空格
- 按大小写去重
- 单标签长度超 `6` 丢弃
- 最多保留 `2` 个标签

## 后台系统设置扩展

- 管理端 `系统设置` 新增“默认帖子标签”输入项
- 支持按换行、逗号或分号分隔多个标签
- 标签清洗规则与发帖一致（去重、去 `#`、单标签最多 `6` 字）
- 最多保留 `50` 个默认标签
- 默认标签是投稿页“公共可选标签池”的唯一来源
- 用户在投稿时临时新增的标签不会进入公共标签池

## 路由约定

- 标签搜索统一落在 `search` 页面，不新增独立标签页
- 统一 URL 形态：
  - `/search?tag=职场`
  - `/search?q=加班&tag=职场`
  - `/search?q=加班&tag=职场&page=2`
